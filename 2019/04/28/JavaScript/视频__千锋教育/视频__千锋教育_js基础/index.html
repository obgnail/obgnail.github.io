Undefined:
当我们试图访问一个不存在的变量时,就会得到一个undefined.除此之外,使用已定义却未赋值的变量也会如此，以为js会自动将变量在初始化之前的值设定为 undefined，而
Undefined类型的值只有一个,那就是undefined.
Null:
只包含一个值的特殊数据类型,所谓的null值,通常是没有值或空值,不代码任何东西,

Null与undefined最大的不同在于,==被赋予Null的变量通常被认为是已经定义了的,只不过他不代表任何东西.==

Infinity:
代表超出JS处理的范围的数字,但是他依然是一个数字.他不是无穷大.

NaN:
表示不是数字,但是它依然属于数字类型.
NaN具有传染性:只要表达式含有NaN,那么最终结果就是NaN

==:值相等,允许类型不相等
===:值相等,类型也要相等

JS与浏览器

BOM:浏览器对象模型(Browser Object Model):
用于访问浏览器和计算机屏幕的对象集合

  1. document:
    页面的文档对象
  2. frames:
    框架集合
  3. navigator:
    浏览器及其功能信息的对象
  4. location:
    控制浏览器地址栏内容,刷新页面,加载页面
  5. history:
    历史记录

事件流:

有一个同心圆，鼠标点击圆心.那么鼠标点击的不是某个特定的圆，而是所有的圆。那么哪个圆应该最先享用这个事件?

我们把事件流分为3个阶段:
时间捕获阶段,事件目标阶段,事件冒泡阶段

捕获阶段和冒泡阶段都可以响应事件.
==监听函数==的第三参数决定了是捕获阶段响应事件还是冒泡阶段响应事件.true为捕获阶段,false为冒泡阶段
